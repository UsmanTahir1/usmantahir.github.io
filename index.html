1. Printing OK

I started with by downloading docker and qemu. Then I made a Dockerfile for describing all the steps to make my enviornment image. This file was run on cmd to build the image. I then ran code on cmd to create an instance of the image i.e the container. Now I had access to the virtual machine. Made a boot loader in header.asm file locate the OS. Then I made main.asm file as the entry point to OS. Then I made linker file to link the OS together followed by grub.cfg file that will create the iso file to host OS. Now i made a Makerfile that hold all build commands for the OS. Finally I ran the OS in Qemu and "OK" was printed.

2. Printing NUST Logo

After accomplishing task 1, I did not face any errors. It was smooth sailing from here and I only had to follow the tutorial. First I modified the previous main.asm file.

  1. Made two lables that reference top and bottom of the stack in .bss section which contains statically allocated variables.
  2. Made function that confirms that OS has been loaded by the mutipli-boot loader.
  3. Made function that checks CPU ID and gives CPU instructions.
  4. Made function that checks for long mode support (64 bit).
  5. We use virtual memory through paging to access 64bit mode.
  6. Made two functions to allow paging. first to setup page table and second to enable paging.

New file main64.asm is created that is externally called by main.asm to enable long mode. Next kernel is built. Now a main.c file is created to use C code in. It is made in kernel subfolder. This code will print NUST logo. It is called in main6 .asm. Now an interface folder is created. which contains print.h file. This defines our printing interface. print.c file is created in x86_64 folder. This describes the characters to be printed.

  1. Clear row function is created to print empty space.
  2. print char function is created to print character at appropriate place.
  3. New line function to move to next line.
  4. print_set_color() function is created for changing the color.

Finally Makerfile is updated to accomodate .c file. Finally I ran the output in Qemu.
